01

整体架构

漫画翻译工具采用分层架构设计，由桌面应用壳、前端UI、Kratos后端服务、外部API和本地存储层五大部分组成。前端通过 HTTP REST 与后端通信，后端编排翻译工作流并调用外部 AI 服务。

02

技术栈

项目采用 Go 后端 + React 前端的技术组合，后端基于 Kratos 微服务框架，工作流使用 langgraphgo 编排，前端使用 Ant Design + Fabric.js 构建。

层级技术说明
桌面壳go-webview2Go 绑定 WebView2，轻量桌面窗口，无需 Electron
前端框架React 18 + TypeScript组件化 UI 开发，类型安全
UI 组件库Ant Design 5企业级 UI 组件，开箱即用
图片标注Fabric.jsCanvas 图形库，支持文字区域选择/拖拽/缩放
后端框架KratosB站开源 Go 微服务框架，HTTP/gRPC 支持
工作流引擎langgraphgoGo 实现的 LangGraph，支持状态图/HITL/断点恢复
数据库SQLite + go-orm轻量嵌入式关系库，零部署
向量存储chromem-goGo 原生向量数据库，嵌入式无外部依赖
LLM 翻译DeepSeek API高性价比 LLM，支持翻译与 OCR
图片加工GPT-image-2 APIOpenAI 图片编辑模型，Inpainting 能力强
构建工具Vite前端快速构建与热更新
包管理Go Modules / pnpmGo 模块管理与 Node.js 包管理
03

翻译工作流设计

翻译工作流基于 langgraphgo 状态图实现，支持自动模式与圈选模式两种 OCR 入口，包含角色自动推断、术语匹配、知识库组装、RAG 检索翻译范例、LLM 翻译、质量校验、用户审核等完整流程，支持 HITL（Human-in-the-Loop）中断恢复。

    文本提取完成 --> 用户选择确认: 自动模式
    文本提取完成 --> 等待用户圈选: 圈选模式
    用户选择确认 --> 角色自动推断
    等待用户圈选 --> 角色自动推断

    术语匹配 --> 知识库组装
    知识库组装 --> RAG检索翻译范例
    RAG检索翻译范例 --> LLM翻译

    LLM翻译 --> 翻译质量校验
    翻译质量校验 --> 用户审核: 通过
    翻译质量校验 --> LLM翻译: 不通过重试

    用户审核 --> 结果确认: 用户接受
    用户审核 --> 用户编辑: 用户修改
    用户编辑 --> 结果确认

    结果确认 --> 更新翻译范例库
    更新翻译范例库 --> 图片加工
    图片加工 --> [*]

05

知识库四层结构

知识库采用四层结构设计，从角色风格定义到术语约束逐层构建翻译上下文，确保每个角色的翻译风格一致性。翻译时按层组装 Prompt，让 LLM 学习“这个角色以前是怎么说话的”。

🎭
第一层 · 角色风格档案
Character Style Profile
为每个角色定义语言特征模板：
  • 人称代词偏好：如自称“老子”/“俺”/“在下”
  • 句式特点：短句祈使句/长句优雅/古风正式
  • 语气词：如“嘿嘿”/“呐”/“哼”
  • 口头禅：角色标志性台词
  • 用词等级：粗俗口语/书面优雅/中性
  • 错误模式：如故意用错字模拟粗鲁
📜
第二层 · 风格规则库
Style Rules
按作用域分层管理翻译风格约束：
  • 作品级规则：整体语言风格、时代背景、读者群定位
  • 群体级规则：种族/阵营/职业的统一说话风格
  • 角色级规则：单个角色的特殊规则（最高优先级）
优先级：角色级 > 群体级 > 作品级
📖
第三层 · 翻译范例库
Few-shot Examples
按角色组织的历史翻译对，作为 Few-shot 示例注入 Prompt：
  • 按角色分组的历史翻译对
  • RAG 向量检索相似原文，注入 Prompt 作为 Few-shot 示例
  • 让 LLM 学习“这个角色以前是怎么说话的”
📚
第四层 · 术语库
Glossary
多语言术语对照（已有的 GlossaryEntry/GlossaryTranslation 结构）：
  • 源术语 → 多目标语言翻译对照
  • 支持全局术语与项目级术语
  • 翻译时强制匹配替换

示例：战锤兽人角色风格

层级字段示例值
L1 角色风格档案角色名哥巴德·铁颚 (Gorbad Ironclaw)
群体兽人 (Orc)
人称代词“俺”/“老子”
句式特点短句祈使句、粗暴直接
语气词/口头禅“WAAAGH!”/“哼！”
用词等级粗俗口语
L2 风格规则群体规则兽人种族：不用书面语、多用感叹号、省略助词
L3 翻译范例Few-shot"I'll crush ya!" → “俺要踩扁你！”
L4 术语库术语"WAAAGH!" → “哇嗷！”（不可翻译为其他词）

知识库 Prompt 组装流程

翻译时，工作流引擎逐层查询知识库，按优先级组装 Prompt 上下文发送给 LLM。“知识库组装”步骤的含义是：加载角色风格档案 → 加载风格规则 → 准备 Prompt 上下文。

sequenceDiagram
    participant W as 工作流引擎
    participant CP as 角色风格档案
    participant SR as 风格规则库
    participant TE as 翻译范例库
    participant GL as 术语库
    participant LLM as DeepSeek

    W->>CP: 查询角色风格档案(character_id)
    CP-->>W: 人称/句式/语气词/用词等级
    W->>SR: 查询风格规则(project+group+character)
    SR-->>W: 作品规则+群体规则+角色规则(按优先级)
    W->>TE: RAG检索相似翻译范例(原文embedding)
    TE-->>W: Top-K历史翻译对(same character)
    W->>GL: 匹配术语
    GL-->>W: 术语替换列表

    W->>LLM: 组装Prompt发送
    Note over W,LLM: System: 角色风格+风格规则
Few-shot: 历史翻译范例
Context: 术语约束
User: 原文
    LLM-->>W: 翻译结果

核心交互流程

以下时序图展示了用户从创建项目到完成翻译的核心交互流程，包括前后端通信、工作流编排和用户审核环节。

sequenceDiagram
    participant U as 用户
    participant FE as 前端(React)
    participant BE as 后端(Kratos)
    participant WF as 工作流引擎
    participant DS as DeepSeek API
    participant RAG as RAG知识库
    participant IMG as 图片加工

    U->>FE: 创建项目/导入图片
    FE->>BE: POST /api/projects
    BE-->>FE: 返回项目信息

    U->>FE: 点击"开始翻译"
    FE->>BE: POST /api/translate/start
    BE->>WF: 启动翻译工作流

    WF->>DS: OCR识别请求
    DS-->>WF: 返回文本区域

    WF->>BE: 返回OCR结果(状态:等待审核)
    BE-->>FE: 推送OCR结果
    FE-->>U: 显示识别的文字区域

    alt 自动模式
        U->>FE: 确认OCR结果
    else 圈选模式
        U->>FE: 圈选/调整文字区域
        FE->>BE: PUT /api/regions/{id}
    end

    FE->>BE: POST /api/translate/confirm
    BE->>WF: 继续工作流

    WF->>RAG: 术语匹配+上下文检索
    RAG-->>WF: 返回匹配术语+上下文

    WF->>DS: LLM翻译请求
    DS-->>WF: 返回翻译结果

    WF->>WF: 质量校验
    alt 校验通过
        WF->>BE: 返回翻译结果(状态:待审核)
        BE-->>FE: 推送翻译结果
        FE-->>U: 显示原文/译文对照
    else 校验不通过
        WF->>DS: 重新翻译
    end

    U->>FE: 审核/修改翻译
    FE->>BE: PUT /api/translations/{id}
    BE->>WF: 确认翻译结果

    WF->>RAG: 更新知识库
    WF->>IMG: 图片加工(Inpainting+渲染)
    IMG-->>WF: 返回加工后图片

    WF->>BE: 工作流完成
    BE-->>FE: 推送完成通知
    FE-->>U: 显示翻译后图片

术语库添加流程

用户可以在翻译过程中随时将原文中的术语添加到术语库，支持划选文字快速入库，一个源术语可对应多条目标语言翻译。

sequenceDiagram
    participant U as 用户
    participant FE as 前端(React)
    participant BE as 后端(Kratos)

    U->>FE: 在翻译面板原文上划选文字
    FE-->>U: 右键弹出菜单(“加入术语库”)
    U->>FE: 点击“加入术语库”
    FE-->>U: 弹出术语编辑弹窗
    Note over U,FE: 弹窗内容：
• 源术语：只读，自动填入划选文字，标注源语言
• 目标翻译：默认显示项目目标语言输入行
• “+”按钮：动态添加更多目标语言翻译行
• 备注：可选填
    U->>FE: 填写目标翻译并确认
    FE->>BE: POST /api/glossary/entries
    BE-->>FE: 返回术语条目(GlossaryEntry + GlossaryTranslation)
    FE-->>U: 提示“术语已添加”
