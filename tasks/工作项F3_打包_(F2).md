## 工作描述

构建单体可执行文件：前端build→go:embed嵌入→Go编译→单exe打包。

具体工作：
1. 前端执行`npm run build`生成dist/目录（HTML/JS/CSS静态资源）
2. 配置go:embed嵌入web/dist/目录，将前端静态资源编译进Go二进制
3. Go交叉编译：目标平台windows amd64
4. 验证单exe文件可独立运行：启动Kratos HTTP Server并提供前端页面
5. 配置应用图标：Windows exe图标资源
6. 创建构建脚本(Makefile/build.sh)：一键构建前后端
7. 测试首次启动体验：双击exe→自动打开浏览器/go-webview2窗口

## 技术要点

- 前端构建：`cd web && npm run build`，输出到web/dist/目录，Vite构建配置需指定base路径
- go:embed嵌入：
  ```go
  //go:embed web/dist/*
  var webDist embed.FS
  ```
  使用Kratos的静态文件服务中间件或`http.FileServer(http.FS(webDist))`提供前端资源
- Go编译：
  ```bash
  CGO_ENABLED=0 GOOS=windows GOARCH=amd64 go build -o comic-translator.exe ./cmd/server
  ```
  注意使用`CGO_ENABLED=0`因为modernc.org/sqlite是纯Go无需CGO
- 静态资源路由：Kratos Server配置fallback路由，API请求走Kratos路由，其他请求fallback到embed的静态文件
- 应用图标：使用rsrc或go-winres工具将.ico文件嵌入exe资源
- 构建脚本：Makefile包含`build-frontend`、`build-backend`、`build-all`目标
- 首次启动：exe启动Kratos HTTP Server监听localhost:8080，go-webview2加载`http://localhost:8080`展示界面
