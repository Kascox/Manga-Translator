## 工作描述

实现术语匹配算法：精确匹配、模糊匹配（子串/编辑距离）、优先级排序（项目级>全局）、批量文本术语标注。

具体工作：
1. 实现 `ExactMatch`（精确全文匹配）：
   - 完全匹配源术语文本
   - 大小写不敏感匹配
2. 实现 `FuzzyMatch`（模糊匹配）：
   - 子串匹配：源术语是否出现在文本中
   - 编辑距离匹配：使用Levenshtein距离，阈值可配置（默认≤2）
   - 返回匹配到的术语及匹配位置
3. 实现 `PriorityMerge`（优先级合并）：
   - 项目术语覆盖全局术语冲突
   - 非冲突术语累加
4. 实现 `BatchAnnotate`（批量文本术语标注）：
   - 输入：文本列表 + 术语库
   - 输出：每个文本中匹配到的术语及位置标注
   - 标注信息：术语ID、匹配文本、位置区间、匹配类型（exact/fuzzy）
5. 实现术语替换建议生成：
   - 根据匹配到的术语，生成原文→译文的替换建议
   - 支持占位符格式（如 `{{TERM:WAAAGH!}}` ）用于Prompt注入

## 技术要点

- 精确匹配优先于模糊匹配，同类型匹配按优先级排序（项目级>全局）
- Levenshtein编辑距离算法实现需优化性能，避免O(n*m)对长文本的暴力计算
- 子串匹配使用 `strings.Index` 实现，注意大小写处理
- 术语匹配位置需返回字符偏移区间，供前端高亮显示
- 批量标注需处理术语重叠（长术语优先，避免短术语截断长术语）
- 替换建议生成需考虑目标语言：根据当前翻译的目标语言查找 `GlossaryTranslation`
