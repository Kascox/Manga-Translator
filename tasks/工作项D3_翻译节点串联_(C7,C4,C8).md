## 工作描述

实现翻译流水线的三个核心节点串联：术语匹配节点→RAG检索节点→LLM翻译节点→质量校验节点，以及知识库组装逻辑。

具体工作：
1. **GlossaryMatchNode**：调用C7术语匹配模块，在原文中匹配术语库术语，标注需强制翻译的条目，输出匹配结果列表
2. **RAGRetrieveNode**：调用C4检索模块，对原文文本生成embedding，在chromem-go中按角色分组检索相似翻译范例，取Top-K（默认K=5）
3. **KnowledgeAssemblyNode**：按四层结构逐层查询知识库组装翻译Prompt上下文（角色风格档案→风格规则→术语约束→Few-shot范例）
4. **TranslateNode**：组装完整Prompt（System Prompt + Few-shot Examples + Context + User原文），调用LLM进行翻译
5. **QualityCheckNode**：调用C8校验模块，对翻译结果进行术语一致性、完整性、角色风格、格式正确性校验
6. 定义条件边：质量校验通过→下一步（用户审核），不通过→重试TranslateNode（最多3次，附加前次翻译及问题说明）

## 技术要点

- 术语匹配算法：全局术语匹配→项目术语匹配→模糊匹配（Levenshtein），优先级项目级>全局级、精确>模糊
- RAG检索：原文→embedding→chromem-go按角色分组检索→Top-K排序→格式化为Few-shot示例，相似度阈值0.7
- Prompt组装结构：System Prompt（角色风格+风格规则+术语约束）+ Few-shot Examples（历史翻译范例）+ Context（前后文）+ User（待翻译原文）
- 质量校验维度：术语一致性（规则匹配）、完整性（长度/关键词比对）、角色风格（规则+LLM评估）、格式正确性（规则检查）
- 重试策略：最多3次，每次Prompt中附加前次翻译及问题说明，3次仍不通过则降低质量要求标记待人工审核
- LLM调用参数：temperature=0.3、max_tokens=1024、top_p=0.9
