## 工作描述

前后端API对接、完整翻译流程联调(OCR→翻译→图片加工)、WebSocket/SSE实时状态推送、错误处理。

具体工作：
1. 对接Project API：创建项目、打开项目、扫描目录
2. 对接Translation API：OCR提取、批量翻译、确认翻译、编辑翻译、重新翻译
3. 对接Glossary API：术语CRUD、导入导出
4. 对接Knowledge API：角色管理、风格规则管理、翻译范例
5. 对接Settings API：获取/更新配置
6. 实现实时状态更新：通过WebSocket/SSE接收工作流节点状态变更推送，更新翻译进度
7. 完整流程联调：选图→OCR→标注角色→翻译→确认→图片加工的端到端流程
8. 错误提示与重试交互：网络错误toast+重试、业务错误中文提示、工作流超时检测

## 技术要点

- API对接方式：前端services/目录下的各service模块调用后端Kratos HTTP API
  - Project API：`/api/projects` (POST创建/GET列表)、`/api/projects/{id}` (GET详情/DELETE删除)
  - Translation API：`/api/translate/start` (POST启动)、`/api/translate/confirm` (POST确认)、`/api/translations/{id}` (PUT编辑)
  - Glossary API：`/api/glossary/entries` (GET列表/POST创建)、`/api/glossary/entries/{id}` (PUT更新/DELETE删除)
  - Knowledge API：`/api/characters` (CRUD)、`/api/style-rules` (CRUD)
  - Settings API：`/api/settings` (GET获取/PUT更新)
- 实时状态推送：
  - 方案一：WebSocket连接`/api/ws`，接收工作流状态变更消息
  - 方案二：SSE连接`/api/events`，接收事件流
  - 前端使用`useWebSocket`自定义Hook管理连接和消息处理
- 完整联调流程：前端选图→调用Translation API启动工作流→接收OCR结果推送→前端展示区域标注→用户确认→接收翻译结果推送→用户审核→确认翻译→接收图片加工完成推送
- 错误处理：
  - 网络错误：全局toast提示，支持重试按钮
  - API业务错误：根据响应code字段展示对应中文提示
  - 工作流超时：前端轮询+WebSocket超时检测
  - 图片加载失败：降级显示占位图，支持重新加载
