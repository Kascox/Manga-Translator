## 工作描述

实现设置管理组件(SettingsModal)：API Key配置、模型选择、全局术语库管理入口、角色管理弹窗、风格规则弹窗。

具体工作：
1. 实现设置入口按钮（左下角齿轮图标），点击弹出SettingsModal
2. 实现API Key配置表单：DeepSeek API Key、GPT-image API Key（掩码显示，如sk-****xxxx）
3. 实现模型选择下拉：DeepSeek（deepseek-chat/deepseek-reasoner）、GPT-4等
4. 实现全局术语库管理界面：术语列表展示、新增/编辑/删除术语条目、导入/导出功能
5. 实现角色管理弹窗（CharacterModal）：角色卡片列表、新增角色、编辑角色属性（人称代词/句式/语气词/口头禅）、合并角色、删除角色
6. 实现风格规则弹窗（StyleRuleModal）：按作用域分组展示（作品级/群体级/角色级）、新建规则、优先级调整、规则预览
7. 设置持久化：调用Settings API保存和加载配置

## 技术要点

- SettingsModal：使用Ant Design `Modal`+`Form`+`Tabs`组件，按分组(Tabs)展示不同设置项
- API Key掩码：显示时仅展示末4位，编辑时使用`Input.Password`组件
- 术语库管理：使用Ant Design `Table`组件展示术语列表，支持搜索过滤、分页
- 角色管理弹窗：角色以卡片形式展示（Ant Design `Card`），按出场频率排序
  - 新建/编辑：表单包含角色名称、群体、人称代词、句式特点、语气词、口头禅、用词等级
  - 合并：选择两个角色合并，保留主角色的属性
  - 删除：删除后需重新分配该角色的翻译条目
- 风格规则弹窗：使用Ant Design `Collapse`按作用域分组
  - 新建规则：选择作用域+填写规则内容
  - 优先级调整：拖拽排序同一作用域内的规则
  - 规则预览：展示规则在Prompt中的组装效果
- 设置持久化：调用`useSettingsStore.updateSettings()`和`loadSettings()`，与后端Settings API对接
