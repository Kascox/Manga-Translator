## 工作描述

实现YAML配置加载（`internal/conf/`）、支持全局配置（`configs/config.yaml`）和工作区配置（`workspace.yaml`）、配置合并逻辑（工作区覆盖全局）、环境变量覆盖。

具体工作：
1. 定义配置结构体（`internal/conf/conf.go`）：
   - `ServerConf` — HTTP监听地址、超时时间
   - `DataConf` — 数据库驱动、DSN
   - `VectorStoreConf` — 向量存储类型与路径
   - `LLMConf` — Provider、API Key环境变量名、模型、Base URL
   - `ImageProcessingConf` — 图片加工Provider与API配置
   - `LoggingConf` — 日志级别与格式
2. 实现YAML文件读取与反序列化
3. 实现配置合并逻辑（使用 `mergo` 库）：
   - 加载默认值（代码内硬编码）
   - 加载全局配置（`~/.comic-translator/config.yaml`，可选）
   - 加载工作区配置（`workspace.yaml`，可选）
   - 高优先级覆盖低优先级
4. 实现环境变量覆盖（`COMIC_TRANSLATOR_` 前缀）
5. 创建默认 `configs/config.yaml` 模板

## 技术要点

- 配置结构体使用 `mapstructure` tag 支持环境变量映射
- `mergo` 库实现配置合并，高优先级非零值覆盖低优先级
- 环境变量前缀 `COMIC_TRANSLATOR_`，如 `COMIC_TRANSLATOR_LLM_API_KEY`
- API Key等敏感信息通过 `api_key_env` 字段指定环境变量名，间接读取
- 工作区配置（`workspace.yaml`）位于项目目录下，随项目分发
- 全局配置位于 `~/.comic-translator/config.yaml`，用户级个性化
